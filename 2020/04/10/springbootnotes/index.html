Spring Boot

Spring Boot 是一个快速开发框架，可以迅速搭建出一套基于 Spring 框架体系的应用，是 Spring Cloud 的基础。

Spring Boot 开启了各种自动装配，从而简化代码的开发，不需要编写各种配置文件，只需要引入相关依赖就可以迅速搭建一个应用。

简单理解Spring Boot 是一个集成了 Spring 各种组件的快速开发框架。

Spring Boot 整合 JSP

Spring Boot 的特点

  • 使用Spring引导页面可以在几秒内构建一个项目
  • 方便对外输出各种形式的服务：REST API、WS、Web、Streaming、Tasks
  • 非常简洁的安全策略集成
  • 支持关系数据库和非关系数据库
  • 支持运行期间内嵌容器，Tomcat、Jetty
  • 自动管理依赖

Spring Boot 整合 HTML

Spring Boot 可以结合 Thymeleaf 模版来整合 HTML，使用原生的 HTML 作为视图。

Thymeleaf 模版是面向 Web 和独立环境的 Java 模版引擎，能够处理 HTML、XML、JavaScript、CSS 等。

整合HTML是整体输出，而使用JSP是一个条目一个条目读取输出，因此效率不如前者。

  • pom.xml
  • appliction.yml
  • Handler

浏览器访问 http://localhost:8181/index/index

但是此方法无法直接访问http://localhost:8181/index.html,Spring Boot 经过扫描之后才能访问静态资源，tempalates中的资源不会自动扫描，只能通过事务解析器的方式后台映射，在static文件夹中的内容可以直接访问

访问效果

解决Spring Boot Application in default package：
① main 函数直接放在了java 目录之下，当放在java目录下的 package目录；
② @springbootApplication 注解失效的情况下，推荐使用@CompentScan 和@EnableAutoConfiguration进行代替；

如果希望客户端可以直接访问 HTML 资源，将这些资源放置在 static 路径下即可，否则必须通过 Handler 的后台映射才可以访问静态资源。
